# Диагностика

## Проверяйте по порядку

1. Тип и источник профиля.
2. Выбранное ядро.
3. Создание фактического конфига.
4. Запуск процесса ядра.
5. Локальные порты или TUN.
6. DNS.
7. Маршрутизацию.
8. Kill Switch.

## Что приложить к сообщению об ошибке

- сборку SG Client;
- тип профиля без секретов;
- режим подключения;
- последние относящиеся к ошибке строки журнала;
- шаги воспроизведения.

Не публикуйте UUID, auth, приватные ключи, полные подписки и другие секретные значения.
