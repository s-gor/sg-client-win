# TUN, DNS и маршрутизация

## TUN

TUN создаёт системный сетевой интерфейс и позволяет направить трафик Windows через выбранный профиль. Для работы требуются права администратора.

Главная кнопка SG Client должна реально включать и отключать TUN, а не только менять визуальное состояние.

## Системный режим

Используйте системный режим для первой проверки: основной трафик Windows идёт через выбранное подключение. Это самый понятный способ убедиться, что профиль работает.

## Раздельная маршрутизация

Раздельный TUN нужен, когда через подключение должны идти только отдельные направления, приложения или наборы адресов.

После изменения правил:

1. переподключитесь;
2. проверьте сайт, который должен идти через туннель;
3. проверьте сайт, который должен идти напрямую;
4. сравните внешний IP;
5. откройте журнал.

## GeoIP, GeoSite и SRS

Portable включает базы, которые использует маршрутизация SG Client:

- `geoip.dat`;
- `geosite.dat`;
- `bin\srss\*.srs`;
- `sg-routing-rules-manifest.json`.

Не заменяйте их случайными файлами из других программ. Версии наборов должны быть согласованы с логикой клиента.

## DNS через TUN

Этот режим направляет DNS в контур подключения. Он полезен, когда доменные правила и защита от DNS-утечек важнее использования локального DNS.

Если домены перестали открываться:

1. отключите DNS через TUN;
2. переподключитесь;
3. проверьте профиль в базовом режиме;
4. затем повторно включите DNS и посмотрите журнал.

## MTU

Слишком большой MTU может приводить к частичной загрузке сайтов, зависанию HTTPS или проблемам с отдельными сетями. Слишком маленький MTU снижает эффективность.

Не меняйте MTU без симптомов. После изменения проверяйте несколько сайтов и загрузку файлов, а не только один ping.
