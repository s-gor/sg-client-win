# Kill Switch и аварийное восстановление

## Назначение

Kill Switch блокирует трафик, который мог бы выйти в интернет вне активного защищённого подключения.

## Перед включением

1. Проверьте профиль без Kill Switch.
2. Убедитесь, что подключение и отключение работают.
3. Запомните расположение `RESET-KILL-SWITCH.cmd`.
4. Только затем включайте защиту.

## Нормальная работа

При активном подключении разрешается трафик, необходимый выбранному режиму. При разрыве соединения нежелательный прямой выход блокируется.

## Аварийное снятие правил

Если SG Client уже закрыт, а обычный интернет не восстановился:

1. откройте папку Portable;
2. запустите `RESET-KILL-SWITCH.cmd` от имени администратора;
3. дождитесь завершения;
4. повторно проверьте обычное соединение Windows.

Скрипт предназначен только для восстановления сетевых правил SG Client. Не используйте его как обычный способ отключения работающего профиля.

## Диагностика

Если блокировка повторяется:

- проверьте, не запущена ли вторая копия SG Client;
- завершите подключение штатной кнопкой;
- посмотрите последние строки журнала;
- проверьте состояние TUN;
- перезапустите клиент и повторите базовый тест без Kill Switch.
