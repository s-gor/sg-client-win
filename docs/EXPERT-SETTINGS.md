# Экспертные настройки

## Для кого предназначен раздел

Обычному пользователю профиль из SG-Panel или подписки обычно не нужно редактировать. Экспертный раздел предназначен для диагностики, сторонних профилей, локальных копий и нестандартных конфигураций.

## Источник параметров

SG Client должен показывать происхождение значения: профиль, подписка, SG-Panel, DNS, PEM или ручной ввод.

## REALITY

Показываются serverName, public key, short ID, fingerprint, Flow/Vision и поддерживаемый FinalMask. ECH, PEM и обычный TLS-pinning для REALITY не применяются.

## TLS

По умолчанию используется системное хранилище доверия Windows. Дополнительно доступны проверка по имени, SHA-256 pinning, пользовательская CA/PEM и ECH.

## XHTTP и RAW/TCP

Интерфейс должен показывать только параметры, используемые выбранным транспортом. Неактивные поля не должны считаться проверенными.

## Три уровня проверки

1. синтаксис и формат;
2. семантика — используется ли поле выбранным режимом;
3. тест полного `config.json` выбранным ядром.

Успешный тест ядром подтверждает принятие конфига, но не гарантирует смысл случайного значения, которое не попало в фактическую конфигурацию.

## Кнопки

- **Проверить** — семантическая проверка и тест ядром;
- **Сохранить** — повторная проверка и запись;
- **Создать копию** — независимый локальный профиль;
- **Отменить** — отбросить несохранённые изменения;
- **По умолчанию** — комплектные значения SG Client;
- **Откатить** — восстановить предыдущую рабочую версию или резервную копию.
