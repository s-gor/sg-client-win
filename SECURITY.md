# Безопасность

## Сообщение об уязвимости

Не публикуйте чувствительные данные в открытых Issue и Discussion.

Перед отправкой отчёта удалите:

- приватные ключи;
- UUID и auth действующих профилей;
- токены подписок;
- адреса, которые не должны быть публичными;
- полные серверные конфигурации;
- неочищенные журналы.

В отчёте укажите:

- версию SG Client;
- версию Windows;
- тип профиля;
- используемый движок;
- последовательность действий;
- очищенный фрагмент журнала.

Для аварийного снятия сетевой блокировки используйте `RESET-KILL-SWITCH.cmd` из portable-пакета.
